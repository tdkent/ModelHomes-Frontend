# Model Homes of the Golden Gate International Exposition, 1939-40

This documentation is for the **React frontend** of the content-driven website [ggiemh.com](https://ggiemh.com) chronicling the history of the model homes built in the San Francisco Bay Area for the 1939-40 World's Fair.

## Tech Stack

- Language: `TypeScript`
- UI Library: `React`
- Components & Styling: `Radix` w/ `shadcn`, `Tailwind CSS`
- Routing Library: `React Router`
- Fetch Library: `Tanstack Query`
- Image Processing: `sharp`
- Lint &amp; Format: `Biome`, `Husky` (pre-commit)
- Testing: `Vitest`, `React Testing Library`
- Build Tool: `Vite`
- CI: `GitHub Actions`
- Cloud Deploy: `S3`, `CloudFront`, `Route 53`, `ACM`

## Testing

UI testing is carried out using Vitest and RTL. Integration test coverage includes high-level routing and page rendering. Tests ensure basic React Router functionality and ability of users to access and correctly use top-level navigation elements from anywhere in the application. Unit tests are limited to the home sorting feature.

## Deployment

`ggiemh.com` is a static frontend hosted on S3 from build files generated by Vite. Build files and image assets are managed in separate S3 buckets. A CloudFront distribution routes requests between the build and assets buckets as needed. CloudFront automatically caches resources to increase response time and avoid repeat direct requests to S3. 

### SSL

The site domain `ggiemh.com` is managed by Route 53. SSL certification is configured for the domain with AWS Certificate Manager.

### Request path

```
Browser
|
-> Route 53 w/SSL
   |
   -> CloudFront distro
      |
      -> S3 (build bucket) -> Fetch build files
      |
      -> S3 (asset bucket) -> Fetch images
```

## UI

### HTML Parsing

A portion of website content is fetched from the database as strings with HTML tags. Each string is sanitized using `dompurify` (technically not necessary since the strings do not contain unsafe tags and do not allow user-generated input, but still good practice) then parsed for rendering using `html-react-parser`. `<a>` tags are parsed and replaced with React Router `Link` components to utilize client-side routing. 

### Images

Each of the 28 model homes on `ggiemh.com` are represented by one or more images. Each image has been processed by `sharp`, a Node.js image processing library, into multiple `.avif` and `.webp` files of different image sizes (480px, 768px, 1280px, and 1920px), and a single 1280px `.jpeg` fallback . This redundancy allows the user's browser to fetch the most efficient format (`.avif` being the most efficient and `.jpeg` being the least efficient) at the size that best matches their device.

#### The `picture` element

Images are fetched responsively using the HTML `picture` element, which allows for alternate versions of the same image to be selected by the browser for display. A custom React component determine the needed image and to construct `sizes` and `srcset` attributes.
