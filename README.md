# Model Homes of the Golden Gate International Exposition, 1939-40

This documentation is for the **React frontend** of the content-driven website [ggiemh.com](https://ggiemh.com) chronicling the history of the model homes built in the San Francisco Bay Area for the 1939-40 World's Fair.

## Tech Stack

- Language: `TypeScript`
- UI Library: `React`
- Component Library: `Radix` via `shadcn`
- Routing Library: `React Router`
- Fetch Library: `Tanstack Query`
- Image Processing: `sharp`
- Lint &amp; Format: `Biome`, `Husky` (pre-commit)
- Testing: `Vitest`
- Build Tool: `Vite`
- CI: `GitHub Actions`
- Cloud Deploy: `S3`, `CloudFront`

## Testing

Testing is generally kept simple, with assertions such as "does the current route load the correct page?" and "is the user able to access and use navigation elements?".

## Deployment

`ggiemh.com` is a static frontend hosted on S3 from build files generated by Vite. Build files and image assets are managed in separate S3 buckets. These buckets are managed by a CloudFront distribution that routes requests to `ggiemh.com` and `ggiemh.com/assets/*` to fetch resources. CloudFront will send cached resources if they are available at the given edge location, or directly from S3 if they are not yet cached.

### Request path

Browser -> Route 53 -> CloudFront -> S3 (build bucket) -> Load from static build files
                       |
                       |
                       --> S3 (image asset bucket) -> fetch images

## Images

Each of the 28 model homes on `ggiemh.com` are represented by one or more images. Each image has been processed by `sharp`, a Node.js image processing library, into multiple `.avif` and `.webp` files of different image sizes (480px, 768px, 1280px, and 1920px), and a single 1280px `.jpeg` fallback . This redundancy allows the user's browser to fetch the most efficient format (`.avif` being the most efficient and `.jpeg` being the least efficient) at the size that best matches their device.
